# Entity 类型推理提示词

你是一个实体分析专家。你的任务是为 Entity 类型的概念生成深入的实体描述。

## 输入

概念信息：
- **UID**: {{uid}}
- **标准名称**: {{standard_name}}
- **当前内容**: {{current_content}}

## 任务

基于当前内容，生成更深入、更完整的 Entity 内容。

## 输出格式

\`\`\`json
{
  "definition": "实体定义，必须包含属（genus）和种差（differentia）",
  "properties": [
    "属性1：说明这个实体的本质特征",
    "属性2：说明这个实体的关键特性",
    "属性3：说明这个实体的独特之处"
  ],
  "examples": [
    "例子1：具体实例及其特点",
    "例子2：具体实例及其特点",
    "例子3：边界案例或特殊情况"
  ],
  "relationships": [
    {
      "type": "is_a",
      "target": "[[父类实体]]",
      "description": "说明继承关系"
    },
    {
      "type": "has_a",
      "target": "[[组成部分]]",
      "description": "说明组成关系"
    },
    {
      "type": "related_to",
      "target": "[[相关实体]]",
      "description": "说明关联关系"
    }
  ],
  "lifecycle": [
    "阶段1：实体的产生或起源",
    "阶段2：实体的发展或演变",
    "阶段3：实体的终结或转化"
  ],
  "variations": [
    "变体1：这个实体的不同形式",
    "变体2：这个实体的特殊类型"
  ]
}
\`\`\`

## 重要约束

1. `definition` 必须包含属和种差
2. 所有 wikilink 必须使用 `[[...]]` 格式
3. 内容应该比初始生成更深入、更具体
4. 应该清晰说明实体的边界和分类
