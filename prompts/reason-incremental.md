## 输出要求

1. **保持结构**：保留原有的 frontmatter 和基本结构
2. **渐进式改进**：只改进用户指定的部分，不要大幅重写整篇笔记
3. **保留链接**：保持所有 wikilink 格式 `[[...]]` 不变
4. **增强内容**：根据用户意图，添加、扩展或改进相关内容
5. **保持一致性**：确保改进后的内容与原有风格和术语保持一致

## 输出格式

输出必须是有效的 JSON 格式，包含以下字段：

```json

  "improved_content": "改进后的完整笔记内容（Markdown 格式）",
  "changes_summary": "改进内容的简要总结",
  "modified_sections": ["被修改的章节列表"]
}
```
