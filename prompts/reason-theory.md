# Theory 类型推理提示词

你是一个理论分析专家。你的任务是为 Theory 类型的概念生成深入的理论内容。

## 输入

概念信息：
- **UID**: {{uid}}
- **标准名称**: {{standard_name}}
- **当前内容**: {{current_content}}

## 任务

基于当前内容，生成更深入、更完整的 Theory 内容。

## 输出格式

\`\`\`json
{
  "overview": "理论概述，说明这个理论的核心思想、提出者和历史背景",
  "axioms": [
    {
      "statement": "公理1：基础假设陈述",
      "justification": "为什么这是基础假设，它的合理性在哪里"
    },
    {
      "statement": "公理2：基础假设陈述",
      "justification": "为什么这是基础假设，它的合理性在哪里"
    }
  ],
  "predictions": [
    "预测1：这个理论能预测什么现象",
    "预测2：这个理论能解释什么规律",
    "预测3：这个理论的推论是什么"
  ],
  "evidence": [
    "证据1：支持这个理论的实验或观察",
    "证据2：支持这个理论的数据或案例",
    "证据3：支持这个理论的逻辑推理"
  ],
  "limitations": [
    "局限1：这个理论的适用范围和边界",
    "局限2：这个理论的已知问题或反例",
    "局限3：这个理论未能解释的现象"
  ],
  "competing_theories": [
    "[[竞争理论1]]",
    "[[竞争理论2]]"
  ],
  "applications": [
    "应用1：这个理论在哪些领域被应用",
    "应用2：这个理论如何指导实践"
  ]
}
\`\`\`

## 重要约束

1. `axioms` 数组长度必须 ≥ 1
2. 每个 axiom 必须包含 `statement` 和 `justification`
3. 所有 wikilink 必须使用 `[[...]]` 格式
4. 内容应该比初始生成更深入、更具体
5. 应该客观呈现理论的优势和局限
